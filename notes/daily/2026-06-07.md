## Conceptual ground covered today

Today was also where the foundational parser concepts finally clicked. Worth keeping for tomorrow-Lohit reading this cold:

- **The EBNF ‚Üí parser primitive mapping:** notation operators (`X`, `X | Y`, `X?`, `X*`, `X+`) translate directly to code patterns (`consume`, `peek+branch`, `if match`, `while match`, parse-one-then-`while-match`). Once you see this mapping, recursive descent stops feeling magical.
- **Why a grammar matters even though tokens are linear:** grammars encode ORDER, REQUIRED/OPTIONAL, REPETITION, and CHOICE rules that pure token-walking can't express. They're also the spec the parser implements.
- **Why trees, not stacks/postfix:** downstream layers (optimizer, executor, semantic analyzer) all want addressable, transformable structure. Postfix loses the structure needed for query optimization, which IS tree rewriting in weeks 5-6.
- **Local-capture pattern for dataclass construction:** capture each `consume()`'s return into a local variable, build all field values first, then construct the dataclass at the end with everything passed in. No need for recursion just to gather values.
- **"Jumping the gun" ‚Äî pre-anticipating future complexity is procrastination wearing the costume of preparation.** Day 3 was just the foundation; column lists (day 4), WHERE (day 5), JOINs (day 6) all build incrementally on the same architecture.
